Vấn đề

3 nỗi đau khi RE / spec app cho team

Doc PM truyền thống không scale được khi handoff cho dev / coding agent rebuild.

🛠️ Cách cũ — manual
  • Screenshot tay từng màn
  • Note Excel / Notion rời rạc
  • Không trace được flow đầy đủ
  • 1 dev rebuild = 1 tháng disagreement
🎯 PM ↔ AI scope drift
  • AI capture cái nó nghĩ, không phải cái PM muốn
  • Stuck dead-end → không ai biết để giúp
  • Spec chỉ 1 platform → iOS xin lỗi đọc lại
  • Không có structured review → drift triền miên
🤖 Doc cũ ≠ machine-readable
  • Coding agent grep mò, đọc 50 trang prose tự do
  • Không anchor → không cross-link giữa screen / API / data model
  • AC viết "AI gửi tin ngắn gọn" → không gen test được
  • Không có index → search bằng substring + may rủi
  • Spec không live cùng code → drift theo thời gian
Giải pháp

Spec markdown structured + queryable graph + PM-in-the-loop

Drive 1 device Android, capture mọi screen, sinh ra 5 file markdown / feature + 1 graph JSON queryable + 1 doc tổng app-level platform-agnostic. Engineer + coding agent rebuild được trên cả iOS / Android không cần chạy app gốc.

⚙️ App-agnostic

Mọi giá trị đặc thù app (package, viewport, blocklist, tabs) ở .spec-profile.yml. 1 toolkit dùng cho mọi app — config-driven, không hard-code.

👥 PM-driven, async

3 PM gate (scope / coverage / spec Q&A) chặn AI trượt. Telegram bridge cho PM review remote. Stuck handoff không treo.

🌐 Cross-platform

Layer 5 platform-agnostic (linter cấm Compose, SwiftUI...). Layer 3 Kotlin = illustrative — coding agent dịch sang Swift/TS/Dart.

🤖 Agent-queryable spec graph

Spec graph JSON (nodes + edges + index) + MCP server 8 tool (spec_path, spec_search, spec_reuses...). Coding agent query precise, không grep mò 50 trang md.

Differentiator chính

Spec graph — query, không grep

Honesty

Workflow chuẩn cho 1 lớp app — phần khác có path nhưng chưa develop

✅ Xử được tốt (v1.x hiện tại)
  • Android native (Compose, View XML, mixed)
  • A11y tree đọc được
  • Navigation chuẩn (bottom nav / drawer / tabs)
  • App lành tính với automation
  • PM sẵn sàng tham gia 3 gate
🚧 Có path, chưa tích hợp
  • iOS — kỹ thuật khả thi qua xcrun simctl + Accessibility Inspector + idb. Chưa develop module (~600 dòng + iOS profile schema). Roadmap candidate.
  • Flutter / RN / WebView — a11y tree thưa nhưng có thể nâng parser hiểu WebView role + Hermes a11y (~200 dòng).
  • UX research mode — skip Layer 3 cho competitor research, profile style mới (~150 dòng).

4-Layer Pipeline · 2026-04-21

Spec-First Architecture

Từ reverse-engineering app gốc → spec + design → MCP agent → coding & testing agents downstream

Layer 1 App Spec

Reverse-engineer app Android thật → 3-layer markdown spec

  • Auto-test tool (droidrun Portal + adb) drive 1 device
  • capture.py ghi PNG + JSON + XML per screen
  • nav_graph.json auto-build bằng screen_hash
  • Claude fill 3 file theo today_* reference style
Output artifact
<feat>_observations.md <feat>_spec.md <feat>_feature_spec.md nav_graph.json

Layer 2 Design

Figma file + design system + component library

  • Figma MCP — get_design_context · get_screenshot
  • Code Connect map: Figma component ↔ codebase component
  • Design tokens → CSS variables / Theme
  • Cross-reference anchor với Layer 1 screen spec
Output artifact
figma:design_context design_tokens.json code_connect_map component_refs/

Shared Infra Spec Graph · MCP Agent · Visualizer

Query layer — biến spec markdown + design artifact thành API có thể hỏi được

Spec Graph

Build từ YAML frontmatter của mọi spec/feature/**/*.md

nodes.json edges.json index.json
MCP Server

8 tools exposed qua .mcp.json project-scope hoặc user-scope

spec_show spec_feature spec_path spec_acceptance spec_reuses spec_search spec_list spec_stats
Visualizer + Subagent

Human lookup + haiku-backed subagent cho multi-query research

spec/viz/index.html .claude/agents/spec-graph.md

Layer 3 Implementation

Coding agent query MCP → code implementation theo Clean Arch + Compose

  • Agent đọc spec_feature <feat> → biết toàn bộ screen + block
  • spec_reuses <component> → tái sử dụng thay vì viết mới
  • spec_show <anchor> → lấy API + data model để scaffold
  • Figma MCP song song → lấy visual + Code Connect mapping
Output artifact
:feature:<name>/ :core:<shared>/ ViewModel + UseCase Compose screens

Layer 4 Auto Test

Test agent query MCP → generate & run test theo acceptance criteria

  • spec_acceptance <feat> --unverified → todo list
  • Invariants từ §4 Cross-screen → integration test
  • QA bugs từ §17 → regression test case
  • Re-drive app qua droidrun để verify drift vs spec baseline
Output artifact
unit tests integration tests Espresso / Compose UI test drift_report.md
